# OSGARD mobile (Expo)

Мобильное приложение OSGARD NEW WORLD: экраны создания артефакта, результата, истории, кошелька/рынка TimeCoin и профиля. Использует тот же Express-бэкенд и БД, что и веб-приложение (`../backend`) — артефакты, баланс и история, созданные на вебе и в мобилке, видны одному пользователю (`owner_id`/кошелёк общий), без дополнительной синхронизации.

## Структура вкладок

- **Создать** (`app/(tabs)/index.tsx`) — генерация артефакта (текст/голос).
- **История** (`app/(tabs)/explore.tsx`) — список артефактов с клиентскими фильтрами и пагинацией.
- **Кошелёк** (`app/(tabs)/wallet.tsx`) — баланс по всем валютам (Credits/Shards/Crystals/TimeCoin/USD), покупка/продажа TimeCoin по рынку (`/tc-market/buy`, `/tc-market/sell`), открытие/закрытие стейков (`/stakes`).
- **Профиль** (`app/(tabs)/profile.tsx`) — данные аккаунта, реферальный код, топ-5 лидерборда архитекторов, последние транзакции, выход из аккаунта.

Экономические хуки и API-обёртки лежат в `lib/{tc-market,stakes,wallet,marketplace,leaderboard,transactions}-api.ts` и `hooks/use{TcMarket,Stakes,Wallet,Marketplace,Leaderboard,Transactions}*.ts` — все запросы кэшируются через React Query и переживают офлайн (см. «Оффлайн-кэш» ниже).

## Аутентификация

Полный auth-стек (`app/(auth)/`): вход/регистрация по email или username, восстановление пароля, OAuth через Google/GitHub (`lib/oauth.ts`), биометрическая блокировка (`store/biometricStore.ts`), гостевой режим (`store/guestStore.ts`) и онбординг (`store/onboardingStore.ts`). Токены хранятся в `expo-secure-store`, не в AsyncStorage.

## Запуск

```bash
cd mobile
npm install
npx expo start
```

Требуется запущенный бэкенд: `cd ../backend && npm run dev` (по умолчанию `http://localhost:3003`).

## Переменные окружения

Создайте `.env` (или `.env.local`) в `mobile/`:

```
EXPO_PUBLIC_API_URL=http://<lan-ip>:3003
```

- На физическом устройстве/эмуляторе `localhost` не резолвится в машину разработчика — укажите LAN IP хоста, на котором запущен `backend`.
- Если переменная не задана, используется значение по умолчанию `http://localhost:3003` (подходит только для запуска в браузере/симуляторе на той же машине).

## Голосовой ввод требует Dev Client

Голосовой ввод (`expo-speech-recognition`) **не работает в Expo Go** — нужна сборка с `expo-dev-client`:

```bash
npx expo prebuild
npx expo run:android   # или run:ios
```

В обычном Expo Go остальной функционал (создание артефакта текстом, история, шеринг) работает как обычно — недоступна только кнопка микрофона.

## Известные ограничения бэкенда (учтены в UI как клиентские компромиссы)

- **Нет SSE/поллинга статуса генерации.** `POST /artifacts/generate-ai` — синхронный запрос, отвечает сразу готовым артефактом. Экран прогресса — клиентский псевдо-прогресс (этапы Charging → Generating → Done) поверх одного `await`, не реальный прогресс сервера.
- **Нет параметра темы.** Sci-Fi/Fantasy/Cyberpunk/Steampunk — клиентские пресеты, подмешиваемые в свободный текст `hint`.
- **Нет серверного дневного лимита генераций артефактов.** Счётчик "N/3 сегодня" — информационный, посчитан на клиенте по уже загруженной истории; реальное ограничение — баланс TimeCoin (`AI_GENERATE_COST_TC = 50`).
- **Нет изображения артефакта.** Backend отдаёт только текстовые поля (`description`, `lore`, `aiVisual`). Карточка результата — стилизованная (цвет по редкости, иконка по типу), не фото.
- **Нет серверной пагинации/фильтров.** `GET /artifacts/mine` отдаёт весь список сразу; пагинация (по 20) и фильтры по дате/типу/редкости в экране «История» — клиентские, по уже загруженным данным.

## Оффлайн-кэш

Список артефактов и баланс кошелька кэшируются через React Query + AsyncStorage-persister (`lib/queryClient.ts`), поэтому история и последние данные доступны и без сети — баннер `OfflineBanner` сигнализирует об отсутствии соединения.
