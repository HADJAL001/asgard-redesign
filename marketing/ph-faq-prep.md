# PH-запуск OSGARD — заготовки ответов на вопросы

Черновик для комментариев в день запуска. Ответы ниже сверены с кодом
бэкенда (не додуманы) — там, где сверить не удалось, явно помечено
«нужно уточнить у продукта», вместо того чтобы гадать.

## «Это крипта / блокчейн?»

Нет. TimeCoin — полностью внутренняя виртуальная валюта в SQLite-базе
бэкенда, без блокчейна и без вывода в реальные деньги (в коде
`wallet.routes.ts` конвертация только между внутренними валютами, реального
cash-out эндпоинта нет). Рекомендация: **не ставить тег "Web3 / Crypto"** на
PH — это может привлечь не ту аудиторию и/или модерацию, а по факту продукт
не финансовый. (В `marketing/product-hunt-launch.md` этот тег уже стоит под
вопросом — предлагаю снять его в пользу "Artificial Intelligence" + "Gaming"
+ один запасной, например "Productivity" или "Web App".)

## «Какие AI-модели используются?»

Цепочка провайдеров с фоллбэком: Grok → DeepSeek → детерминированный
локальный генератор без внешних вызовов, если оба провайдера недоступны/не
сконфигурированы (`backend/src/services/ai-artifact-generator.ts`). Это
значит генерация артефакта не падает даже без ключей API — деградирует к
локальному алгоритму, а не выдаёт ошибку пользователю.

## «Как войти — есть ли обычная регистрация email/пароль?»

Да, есть email/пароль-регистрация (`POST /auth/register`), плюс OAuth —
но реально подключены только **Google и GitHub** (в БД зарезервированы
колонки под discord/facebook/twitter, но они не реализованы — если кто-то
спросит про вход через Discord и т.п., честный ответ "скоро", не "сейчас").

## «Есть мобильное приложение? На каких платформах?»

Да — Expo/React Native, iOS + Android, общий бэкенд с веб-версией (один
профиль/кошелёк/история на всех устройствах). На момент запуска приложение
готово к сборке (иконки, EAS-конфиг, метаданные для сторов), но публикация
в App Store/Google Play ещё не выполнена — если спросят "где скачать
мобильное" в день запуска, честный ответ: "в процессе публикации в сторах,
следите за обновлениями" (не обещать конкретную дату, пока билд не собран).

## «Это бесплатно?»

**Нужно уточнить у продукта** — в коде есть признаки более широкой системы
тарифов/квот на генерации (`generationsQuota.ts`, докупка пакетов), но не
проверял детально применительно к самому OSGARD-артефакт-флоу, чтобы не
дать неточный ответ в публичном PH-треде. Рекомендация: перед запуском
кто-то из продукта явно фиксирует один короткий ответ на этот вопрос
(бесплатный лимит N генераций/день, дальше платно — или полностью бесплатно
на старте), чтобы вся команда отвечала одинаково.

## «Опенсорс?»

Не проверял намеренно (не тема моей задачи и не то, что я могу решить
технически) — уточнить у продукта перед публикацией лончкита, чтобы не
обещать репозиторий, которого нет.

(автоскейл/лимиты) с тем, кто его настраивал.
